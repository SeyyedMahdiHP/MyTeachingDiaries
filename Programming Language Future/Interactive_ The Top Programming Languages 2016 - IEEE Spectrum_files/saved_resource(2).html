                                                                                                                                                                          alexander manley David Murphy • a month ago

                                                                                                                                                                          Yes, NoSQL is not a language, however language matters when designing NoSQL systems. Take CockroachDB which is a NoSQL database written in Go that also supports SQL. Given certain limitations around Java for very high performance systems, the designers are going with Go in order to gain C level performance but without the associated complexities and memory management problems/workload. Hence Go and SQL in this case are linked to the aforementioned use of the term "NoSQL".
